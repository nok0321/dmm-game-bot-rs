# テンプレート画像セット

DMM ブラウザゲーム自動化ツール `dmm-game-bot` 用のテンプレート画像です。
クエスト「天地狂乱」(降魔の戦場) を対象としています。

## 切り出し元情報

- **元画像サイズ**: 1277 × 693 px (Chrome ブラウザのスクリーンショット)
- **ゲーム描画領域**: 概ね x=200〜1100, y=130〜660 の範囲
- **環境**: Windows 11 + Chrome (DPI 100% 想定)

切り出し座標は具体的なピクセル位置で固定されています。**運用時のウィンドウサイズが
切り出し元と異なる場合、テンプレートが一致しなくなる可能性があります**。
その場合は同サイズで運用するか、別解像度のテンプレートを追加してください。

## 操作シーケンス (順序主導モデル)

旧版の「画面アンカーで状態判定 → クリック対象を選ぶ」モデルから、
**指定順にクリック対象テンプレを探して押す**順序主導モデルに変更しました。

```
[AP回復フェーズ]
1. ap_plus_button         Home の AP+ アイコンをクリック
2. ap_recovered_use_max   「最大選択」をクリック
   ★ 2.5: 霊晶石ガード (後述) — マッチしなかったら即停止
3. use_button             「使用」をクリック
4. tap_indicator          「TAP」表示で回復モーダルを閉じる

[討伐起動フェーズ]
5. toubatsu_button        Home の「討伐」(朱色) をクリック
6. toubatsu_start         「討伐開始!」をクリック

[周回〜後始末フェーズ]                    ← ステップ 6 の直後から最大 45 分のロング待機
7. next_button            (1回目) battle 終了後に「次へ」
8. next_button            (2回目) もう一度「次へ」
9. close_button           [Optional] 報酬達成モーダルが出たら「閉じる」
                          報酬上限到達後は表示されないため、未検出でも正常

→ 1 に戻ってループ (停止条件は後述)
```

## 周回タイミング・待機戦略

- **ステップ 1〜5, 7, 8**: 検出ポーリング 1〜2 秒間隔、テンプレ未検出が 60 秒続いたら異常停止
- **ステップ 6 → 7**: 討伐開始後はゲーム内で 30 分以上の自動周回が走るため、
  `next_button` を **5〜10 秒間隔で最大 45 分ポーリング**。タイムアウト時は異常停止
- **ステップ 8 → 9 (Optional)**: ステップ 8 後に `close_button` を **最大 30 秒**だけ探し、
  見つからなければ次のループへ進む (報酬上限到達後の正常パターン)
- **next_button のデバウンス**: 7 でクリックした後、画面遷移を待つため
  `next_button` が一旦消えるまで確認してから 8 の検出を始めること
  (同一画像連続クリック誤動作の防止)
- **ループ復帰判定**: ステップ 8 or 9 完了後、再び `ap_plus_button` が検出された時点で
  1 周完了とみなす (Home 専用アンカーは不要)

## 霊晶石ガード仕様

### リスク

「最大選択」は AP 不足分を `英気の霊薬(小→中→大) → 霊晶石` の順に自動充填します。
英気の霊薬の在庫が枯渇したフェーズで `最大選択 → 使用` を押すと、**課金通貨である
霊晶石 (1個=185 AP回復) が自動消費されます**。実機ストックで例えば 13.5 万個保有して
いる場合、誤動作 1 回でも被害が大きい高リスクポイント。

### 対策 (テンプレマッチによるポジティブ確認)

ステップ 2 (`最大選択` クリック) 完了直後、ステップ 3 (`使用` クリック) に進む前に、
**霊晶石スロット (右端 4 列目) の選択数が 0 であること**をテンプレマッチで検証する。

- 必要テンプレ: `reisseki_zero_guard.png`
  - **内容**: 「霊晶石×100」スロット下部の `< 0 >` 数量セレクタが「0」を表示している領域
  - **ROI**: 右端スロットの数量カウンタ部分のみに探索範囲を制限する
    (他スロットの 0 表示と区別するため)
  - **撮影タイミング**: アイテム選択モーダルを開いた直後 (どのアイテムも未選択の初期状態)
- 検証手順:
  1. ステップ 2 (最大選択クリック) 完了
  2. 霊晶石スロットの数量領域に対し `reisseki_zero_guard.png` をマッチ
  3. **マッチした** → 霊晶石は 0 と確認できたので `使用` クリックへ進む
  4. **マッチしなかった** → 霊晶石が選択された可能性 → **絶対に `使用` を押さず即停止し、
     ユーザに通知**

### UI 安定性についての判断

「天地狂乱」のアイテム選択モーダル UI は**リリースから 6.5 年変更されていない**ため、
テンプレマッチでの運用で十分に安定。OCR 等の重い実装は当面不要と判断。
将来 UI が変更された場合は別解像度・別レイアウト対応として `reisseki_zero_guard_v2.png`
等を追加する方針。

## 停止条件

### 通常停止: デイリー切り替え (毎日 05:00 JST)

DMM ゲームのデイリー報酬は **毎日 05:00 JST** に切り替わるため、
このタイミングでループを終了する。

- **判定タイミング**: 各ループの先頭 (ステップ 1 開始前) で現在時刻を確認
- **判定条件**: 現在時刻が `05:00 JST` を超えていればループ終了
- **進行中サイクルの扱い**: ステップ 6 (討伐開始) を踏んだ後は、その回の
  周回・後始末 (ステップ 7〜9) を完走させてから停止する
  (途中停止すると霊薬を消費しただけで報酬を取り損ねる)

### 異常停止

- 各ステップのテンプレ未検出が想定タイムアウトを超えた場合 (上記「周回タイミング」参照)
- 霊晶石ガード (ステップ 2.5) が失敗した場合 — **最優先で停止**
